OpenEdu-VirtualProcessManagement (version v 0.5.5 beta) by residuen


Das Programm

OpenEdu-VirtualProcessManagement ist ein OpenSource-Tool, welches die Simulation einer automatisierten Anlage ermöglicht. Dies kann z.B. ein Lager- oder ein Warenhaus mit Wareneingang und Ausgang sein, oder aber auch eine Fertigungsstraße mit Robotern.
Derzeit besteht die Möglichkeit, Hallen mittels eine CSV-Datei zu generieren, welche Wege für Maschinen und Menschen, Stapler, einfache Fahrzeuge, Ladung bzw. Ware und Lagerflächen beinhalten.

Der HTTP-Server

Eine einfache HTTP-Anfrage an den Server veranlasst z.B. Bewegungen oder Lade-Entladevorgänge:
http://192.168.1.2/client?chargeobjectbyid=43,51,left z.B. veranlasst den Stapler mit der ID 43 Objekt 51 aufzuladen.
Der in Java geschriebene Server läuft entweder auf dem lokalen Rechner, oder man startet den Server auf einem Server-PC auf den dann verschiedene Clients Zugriff haben.

Intention

Ziel ist das Planen und Entwickeln von simulierten Steürungs- bzw. Warenmanagementanlagen. Aufgaben können sowohl selbstständig, als auch in Teams in kooperativen Projektaufgaben durchgeführt werden.
Ein wesentlicher Vorteil bei der Implementierung per HTTP-Server ist, das die Lösung von Aufgaben Programmiersprachenunabhängig ist. Auch die Anbindung einer Datenbank für eine Lagerverwaltung ist als Aufgabe möglich, sowie die Eigenentwicklung einer textbasierten oder auch grafischen Benutzeroberfläche für Steürungs- und/oder Lagerlogistikaufgaben.

Befehlsübersicht

Neben der Möglichkeit, den Server per HTTP zu steuern, existiert eine rudimentäre API, um die gleichen Befehle mittels einfacher Methoden zu übermitteln.

API-BefehlHTTP-BefehlFunktionServer-Antwort
getObjectInfo("getall")"http://localhost/client?objectinfo=getall"Anfrage nach allen ObjektenDetails zu allen Objekten
getObjectInfo("getallstatic")"http://localhost/client?objectinfo=getallstatic"Anfrage nach allen statischen ObjektenDetails zu allen Objekten
getObjectInfo("getallmoveable")"http://localhost/client?objectinfo=getallmoveable"Anfrage nach allen beweglichen ObjektenDetails zu allen beweglichen Objekten
getObjectInfoByGroup(2)"http://localhost/client?objectinfo=getbygroup:2"Anfrage nach allen Objekten einer GruppeDetails zu allen Objekten einer Gruppe
getObjectInfoByGroup(2, 0)"http://localhost/client?objectinfo=getbygroup:2,0"Anfrage nach einem Objekt über Gruppe und IDDetails zu einem Objekt
moveObject(43, "down")"http://localhost/client?moveobject=43,down"Bewegt Objekt 43 um eine Zelle nach untenBestätigung des Steürbefehls
chargeObjectById(43, 51, "left")"http://localhost/client?chargeobjectbyid=43,51,left"Lädt auf Objekt 43 die Ladung/Objekt 51 welches links liegtBestätigung des Steürbefehls
dischargeObjectById(43, "up")"http://localhost/client?dischargeobjectbyid=43,up"entlädt das Objekt 52 das auf Objekt 43 links nach obenBestätigung des Steürbefehls
